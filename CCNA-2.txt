نکته:به لایه های 5,6,7 osi گویند upper layer و لایه های 1,2,3,4 گویند lower layer
یک پرتکل دیگری هست درون TCP که از آن استفاده می کند وقتی که ارتباط sun,syn,ack,syn را انجام می دهد وقتی بسته اول را ارسال می کند و تائیده گرفته می شود می آید دفعه بعد به جای اینکه یک بسته دیگر را ارسال کند می آید دو بسته ارسال می کند همین طور رو به بالا بسته ها را افزایش م یدهد تا اینکه یکی از بسته های ما lost یا گم شود یا از بین برود که می گویند بسته قبلی که گم یا پاک شده را دوباره بفرست همین طور تا به انتها برسد به این فرایند گویند TCP Windowing
ما می توانیم با دستور arp -a تمامی کش های که برای دریافت آدرس dns گرفته ایم را مشاهده نمائیم ما می توانیم با ساتفاده از سرویس dns آیی پی که دریافت می نمائیم 
را با آدرس آی پی دیگر بگیریم:
1 - nslookup google.com
Address:  85.15.1.14
نکته:Address:  85.15.1.14 این ادرسی که دریافت کرده است را از DNS Server همین خود سیستم ما دریافت کرده است که آدرس آن 192.168.1.1 هست
nslookup google.com 4.2.2.4
Address: 4.2.2.4
ولی زمانی که آدرس Dns Server خود را می زنیم از این Dns Server دریافت می نماید
Host ID / NET ID
به سه بخش اول آی پی گویند Net ID و به قسمت آخر آی پی گویند Host ID
192.168.1
255.255.255
NET ID
192.168.1."2"
255.255.255."0"
به بخش " " گویند HOST ID
نکته:به تعداد 1 های subnet mask نشان دهنده net id هستند و 0 ها نشان دهنده host id هستند به صورت باینری
net ID نشان دهنده شبکه ما هست یعنی درون شبکه ما از چه رنج آی پی استفاده می نمائیم به طور کلی net id برای همه شبکه های ما یکسان هست و این host id هست که تغییر می نماید
نکته:ما برای اینکه به دستگاه های خود آی پی host id به دهیم از 0 تا 255 هست که نمی توانیم دوتای آن را به دهیم 0و 255 برای اینکه از broadcast استفاده می شود
در IP v4 ما سه روش داریم برای اینکه ارسال اطلاعات کنیم:
Unicast , brodcast, multicast
unicast :بسته فقط برای یک دستگاه مورد نظر ارسال می شود
multicast:بسته های اطلاعاتی برای گروه ها مشخص ارسال می شود
broadcast:برای همه دستگاه ها ارسال می شود
نکته:یکی از وظایف اصلی روتر همین هست که اجازه نمی دهد بسته های broadcast به بقیه شبکه ها ارسال شود
کلاس های IP:
first bit A:1-126
first bit B:128-191
first bit C:192-223
در کلاس A اولین بیت ما با 0 شروع می شود برای همین هست که subnet mask ما به صورت 255.0.0.0 هست در قسمت net id ما می توانیم تعداد رنج آی پی که دهیم به سیستم ها 126 تا هست و برای اینکه به سیستم های خود آی پی دهیم 24 بیت دیگر می ماند که می توانیم 16777214 تا سیستم هست 
کلاس Bبا بیت های 0 و 1 شروع می شود 16 بیت اول نشان دهنده net id هست 16 بیت آخر نشان دهنده host id درون شبکه ما subnet mask ما 255.255.0.0 هست
و تعداد رنج آپی که می توانیم به دهیم 16384 تا هست و به تعداد سیستم های خود می توانیم 65534 آی پی دهیم 
در کلاس C بیت های ما از 1,1,0 شروع می شود تعداد بیت های net id 24 تا هست و تعدا host id ما 8 تا هست تعداد subnet mask ما 255.255.255.0  هست
و تعداد رنج رنج آی پی که می توانیم دهیم 2097151 هست و تعداد host id های ما 254 تا هست
